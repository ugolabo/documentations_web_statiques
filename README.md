# Documentations avec sites web statiques

Une recension d'outils pour construire des documentations web mises à jour en continu pour :

- le développement et la gestion de projets,
- la documentation techniques et une banque de connaissance,
- le contenu de coaching, d'enseignement de logiciels, d'insertion d'employé,
- la production de rapport et de tableaux de bord.

## Générateurs de site web statiques

Au-delà de l'utilisation des fonctionnalités de GitHub Pages, par exemple, il existe une bonne quantité de <a href="https://jamstack.org/generators/" target="_blank">Site Generators</a>, en differents langages et pour différents usages.

<img src="img/documentations_web_statiques.png" alt="" width="500px" >

Le générateur sert à développer le contenu et à construire le site en local. Certains générateurs offrent des thèmes alors que d'autres nécessitent l'ajout d'un thème avec un template. Selon le générateur, la diversité des thèmes peut être large pour répondre à différents besoins qui dépassent les documentations en ligne (blogue, CV en ligne, portfolio, etc.). Il reste que tout se personnalise en injectant plus de CSS et de JavaScript. Le résultat peut ensuite être hébergé en ligne avec différents services dont GitHub, GitLab, Atlassian Bitbucket, des fournisseurs cloud, avec ou sans nom de domaine. Certains générateurs offrent l'option de publier en divers formats : HTML, PDF, ePub, Word. Voici des exemples utilisés à de nombreuses occasions :


- <a href="https://about.readthedocs.com/" target="_blank">Read the Docs</a> avec un thème fixe et peu de personnalisation.
- <a href="https://www.gitbook.com/" target="_blank">GitBook</a> avec ses thèmes standards, mais plus de personnalisation.
- <a href="https://about.readthedocs.com/" target="_blank">MkDocs</a> avec ses thèmes standards et indépendants comme <a href="https://squidfunk.github.io/mkdocs-material/" target="_blank">Material</a>. Deux exemples ouverts :
    - https://ugolab.gitlab.io/formation/
    - https://ugolab.gitlab.io/projetNF/
- <a href="https://gohugo.io/" target="_blank">Hugo</a> avec une large collection de <a href="https://themes.gohugo.io/" target="_blank">thèmes</a>  
    - ugofolio
- <a href="https://quarto.org/docs/gallery/" target="_blank">Quarto</a> pour créer des rapports, des articles, des dashboards, des sites web et des blogs.
